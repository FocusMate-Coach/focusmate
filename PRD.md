# PRD (Product Requirements Document)
# FocusMate - AI 기반 초개인화 학습 패턴 분석 코치

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 제품명 | FocusMate |
| 버전 | v1.0.0 (MVP) |
| 개발 기간 | 2일 (공모전 출품용) |
| 플랫폼 | 웹 애플리케이션 |
| 타겟 사용자 | 자기주도학습이 필요한 중·고등학생 및 수험생 |

### 한 줄 설명
> "공부 시간이 아닌 집중도로 학습을 최적화하는 AI 학습 코치"

---

## 2. 문제 정의 (Pain Points)

| # | 페인 포인트 | 현재 상황 |
|---|------------|----------|
| 1 | 비효율적 학습 | 학생들이 절대적 공부 시간에만 집착하고 생체 리듬을 고려하지 않음 |
| 2 | 주관적 자기평가 | 객관적 데이터 없이 "열심히 했다/못 했다"는 감에만 의존 |
| 3 | 피드백 부재 | 어떤 시간대, 어떤 방식이 효율적인지 파악할 수단이 없음 |
| 4 | 번아웃 | 무작정 오래 공부하다 지쳐서 학습을 포기하는 경우 발생 |

---

## 3. 해결 방안 (Solution)

학생이 직접 측정한 **학습 시간 + 집중도 데이터**를 수집하고, AI가 이를 분석하여 **시간대별 최적 학습 방식**을 추천하는 개인 맞춤형 AI 코치를 제공합니다.

---

## 4. 사용자 스토리

### 타이머 기능 (Data Collection)
- `US-01` 학생으로서, 공부 시작 시 과목과 학습 유형을 선택하고 타이머를 시작하고 싶다.
- `US-02` 학생으로서, 타이머를 종료하면 집중도(1~5점)를 직관적인 UI로 평가하고 싶다.
- `US-03` 학생으로서, 학습 세션(시간대, 과목, 유형, 집중도)이 자동으로 저장되기를 원한다.

### 대시보드 기능 (Data Visualization)
- `US-04` 학생으로서, 시간대별 평균 집중도를 히트맵으로 한눈에 보고 싶다.
- `US-05` 학생으로서, 과목별 학습 시간 및 효율을 차트로 비교하고 싶다.
- `US-06` 학생으로서, 최근 일주일간 집중도 추이를 그래프로 확인하고 싶다.
- `US-07` 학생으로서, 오늘의 학습 세션 기록을 목록으로 확인하고 싶다.

### AI 피드백 기능 (AI Feedback)
- `US-08` 학생으로서, 버튼 클릭 한 번으로 내 학습 데이터를 기반한 AI 피드백을 받고 싶다.
- `US-09` 학생으로서, "어느 시간대에 어떤 공부가 효율적인지" 구체적인 추천을 받고 싶다.
- `US-10` 학생으로서, 피드백이 단순 분석이 아닌 행동 유도형(~해보세요!)으로 제공되기를 원한다.

## 5. 핵심 기능 요구사항 (Functional Requirements)

### FR-01: 스마트 학습 타이머
- 과목 선택: 국어, 영어, 수학, 과학, 사회, 기타
- 학습 유형 선택: 개념학습, 문제풀이, 복습, 인강시청, 오답노트
- 스톱워치 방식 타이머 (시:분:초 표시)
- 시작 / 일시정지 / 정지 기능

### FR-02: 집중도 자가 평가 모달
- 타이머 정지 시 자동 표시
- 1~5점 이모티콘 방식 UI (😴 😕 😐 😊 🔥)
- 세션 저장 (타임스탬프, 과목, 유형, 시간, 집중도 포함)

### FR-03: 학습 데이터 대시보드
- 시간대별 집중도 바 차트 (0시~23시)
- 과목별 총 학습 시간 파이/바 차트
- 7일 집중도 추이 라인 차트
- 오늘의 학습 세션 타임라인
- 최초 실행 시 가상 데이터(Mock Data) 자동 주입

### FR-04: AI 맞춤형 피드백
- OpenAI API 연동 (gpt-4o-mini)
- 분석 데이터: 시간대별 집중도 패턴, 과목별 효율, 학습 유형별 성과
- 출력 형식: 강점 분석 + 약점 분석 + 구체적 행동 추천 3가지
- 로딩 중 스켈레톤 UI 표시

## 6. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|----------|
| 반응성 | 타이머 업데이트 ≤ 1초 오차 |
| 로딩 속도 | 초기 페이지 로드 ≤ 2초 |
| 데이터 지속성 | 브라우저 새로고침 후에도 데이터 유지 (localStorage) |
| 접근성 | 모바일/태블릿/데스크탑 반응형 UI |
| AI 응답 | OpenAI API 응답 ≤ 10초 |
